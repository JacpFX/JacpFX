JacpFX

The JacpFX Project is an API to create Rich Clients in MVC style with JavaFX, Spring (or other DI frameworks) and an Actor like component approach.

With JavaFX 8, Java GUI development on Client side becomes (more and more) competitive again. Since the release of JavaFX 2 many frameworks were created to speed up client development with two major goals: "**structuring the application**" and "**avoid blocking UI**". Most of this projects are using a server centric approach to create enterprise grade applications. JacpFX on the other hand is a Rich Client framework with similar goals but, but completely client centric. With JacpFX you structure your application with components, perspectives and a workbench loosely coupled through component id's. JacpFX components and perspectives are simple controller classes which makes it easy to compose you applications out of fxml files and programmatic UI parts. All parts are connected through a message bus and have a defined LiveCycle to separate the task execution and UI changes.
